Technische Dokumentation

Was ist dieses Tool?

Der ZVV GTFS Explorer ist ein MCP-Server (Model Context Protocol), der den gesamten Schweizer ÖV-Fahrplan als strukturierte, abfragbare Datenbank bereitstellt. Er dient als Datenbrücke zwischen KI/LLM-Systemen und dem Schweizer öffentlichen Verkehr.

Primärer Nutzen: AI-Assistenten (Claude, ChatGPT, Copilot etc.) können über das MCP-Protokoll live auf Fahrplandaten zugreifen – Haltestellen suchen, Abfahrten abfragen, Verbindungen analysieren – ohne dass die Daten ins Sprachmodell "eintrainiert" werden müssen.

Wofür kann man es nutzen?

  • AI-gestützte Fahrplanauskunft – LLMs können Fragen wie "Wann fährt der nächste Zug ab Bern nach Zürich?" direkt beantworten
  • Datenanalyse – Welche Haltestellen haben die meisten Verbindungen? Wie ist das Netz aufgebaut?
  • App-Entwicklung – REST-API (POST /api/query) für eigene Anwendungen nutzen
  • Open-Data-Exploration – Freie SQL-Abfragen auf den gesamten Schweizer ÖV-Datensatz
  • Barrierefreiheit & Planung – Haltestellenkoordinaten, Umsteigebeziehungen, Liniennetze analysieren

Datenaufbereitung (Pipeline)

Die Rohdaten werden in drei Schritten automatisch aufbereitet:

  1. Download (download-gtfs.js)
    Die Übersichtsseite auf opentransportdata.swiss wird gescraped (via cheerio), der neueste ZIP-Link identifiziert (nach Datum im Dateinamen sortiert), heruntergeladen (~320 MB) und entpackt.
  2. Import (import-gtfs.js)
    Die GTFS-CSV-Dateien werden zeilenweise geparst und in eine SQLite-Datenbank importiert. Dabei werden Tabellen, Spaltentypen und Indexe automatisch erstellt. Speichereffizient: Zeile für Zeile, kein vollständiges Laden in den RAM.
  3. Serving (server.js)
    Die SQLite-Datenbank wird über einen Express.js-Server bereitgestellt: als MCP-Endpoint (POST /mcp) für KI-Tools, als REST-API (POST /api/query) für das Frontend, und als Health-Check (GET /health) für Monitoring.

MCP-Integration (für KI-Tools)

Der Server implementiert das Model Context Protocol (MCP) über StreamableHTTP-Transport. KI-Clients können sich unter POST /mcp verbinden.

Verfügbare MCP-Tools:

ToolBeschreibung
search_stopsTextsuche nach Haltestellen (Name, Limit)
get_routesLinien filtern nach Typ und/oder Agency
get_departuresAbfahrten ab einer Haltestelle (Stop-ID, Datum, Uhrzeit)
get_trip_detailsAlle Halte einer bestimmten Fahrt
get_agenciesAlle Verkehrsunternehmen auflisten
query_gtfsFreie SQL-SELECT-Abfrage (nur lesend, mit Sicherheitsfilter)

MCP-Ressourcen:

URIBeschreibung
gtfs://statusAktueller Datenstatus, Tabellenstatistiken
gtfs://schemaDatenbankschema aller Tabellen

REST-API (für Frontend & Apps)

EndpointMethodeBeschreibung
/GETFrontend (dieses UI)

// Beispiel: Haltestelle suchen
fetch('/api/query', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({

  })
});
